# 🚀 STEP 1 — Convert Convolution to Systolic Architecture

### Current (likely)

```
for each kernel
   multiply
   accumulate
```

Sequential MAC operations.

### Upgrade → **Processing Elements (PE)**

Each PE does:

```
out = out + (weight × input)
```

Basic **PE structure**

```
module PE(
    input clk,
    input [7:0] a,
    input [7:0] b,
    input [15:0] acc_in,
    output [15:0] acc_out
);

assign acc_out = acc_in + (a * b);

endmodule
```

---

### Build 3×3 Systolic Array

Structure:

```
PE00  PE01  PE02
PE10  PE11  PE12
PE20  PE21  PE22
```

Data flows like a **wave**:

```
Inputs → →
Weights ↓
```

Each clock cycle:

```
new input shifts right
new weight shifts down
```

Result:

* **9 MAC operations per cycle**
* Huge speed improvement

# 🚀 STEP 3 — Confidence Prediction Module

Goal:

```
confidence = highest class probability
```

Simplified hardware logic:

```
max_value = max(activation_map)
sum_value = sum(activation_map)

confidence = max_value / sum_value
```

Simpler hackathon version:

```
confidence = max_value
```

Hardware comparator tree:

```
a > b ? a : b
```

Example:

```
max1 = max(a,b)
max2 = max(c,d)
max_final = max(max1,max2)
```

Resource cost:

```
~30 LUTs
```

# 🚀 STEP 5 — XAI Hardware Module

We want something **simple but convincing**.

Goal:

Find **most important feature location**.

Idea:

```
importance = max activation neuron
```

Logic:

```
scan activation map
find max value
store index
```

Example output:

```
Most active neuron = 14
```

Interpretation:

```
Region 14 contributed most to prediction
```

Hardware:

```
max_value
max_index
```

Example Verilog concept:

```
if(activation[i] > max_val)
begin
   max_val = activation[i];
   max_index = i;
end
```

Resource usage:

```
~40 LUTs
```

Very cheap.
